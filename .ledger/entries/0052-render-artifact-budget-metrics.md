# 0052: Render Artifact Budget Metrics

## Summary

Added configurable static-reader artifact budgets and render metrics. `ledger
render` now reports generated bytes and write time, while `ledger doctor` checks
the generated HTML, search index, graph, total bytes, and write duration against
`render.budgets`.

## Why

Lazy search and graph artifacts make the reader more useful, but they can also
grow quietly. Ledger should keep generated payload cost visible so projects can
catch reader size and token-retrieval growth before it becomes normal.

## Changed Files

### src/types.ts, src/config.ts, src/workspace.ts, .ledger/config.yaml

- What changed: Added `render.budgets` with defaults for HTML bytes, search
  index bytes, graph bytes, total bytes, and write time. New workspace scaffolds
  include those defaults.
- Anchor: `render.budgets`
- On conflict: Keep budget values positive and explicit in config.

### src/render.ts

- What changed: `writeStaticReader` now measures artifact sizes, total bytes,
  write duration, and budget status. Added `checkRenderBudgets` for reuse by
  diagnostics.
- Anchor: `writeStaticReader`, `checkRenderBudgets`
- On conflict: Keep generated artifact metrics derived from files on disk after
  writing, not from estimates.

### src/cli.ts and src/doctor.ts

- What changed: `ledger render` prints budget status, and `ledger doctor`
  reports a `render-budget` check.
- Anchor: `ledger render`, `ledger doctor`
- On conflict: Over-budget output should warn, not fail ordinary adoption.

### README.md and docs

- What changed: Documented render sidecar budgets, budget config, and doctor
  reporting.
- Anchor: `Render Budget Config`
- On conflict: Keep docs clear that budgets apply to generated artifacts only.

### tests

- What changed: Added config, render, doctor, and E2E assertions for budget
  config and render artifact metrics.
- Anchor: `test/config.test.ts`, `test/render.test.ts`, `test/doctor.test.ts`
- On conflict: Preserve tests that budget defaults merge with partial configs.

## Behavior And UX Impact

Users see render budget status immediately after `ledger render`, and `ledger
doctor` includes a compact `render-budget` health check. Projects can override
budgets when larger ledgers intentionally need larger static artifacts.

## Invariants

- Render budgets are advisory diagnostics and should not make Markdown source
  invalid.
- Generated artifact sizes are measured after files are written.
- Budget config values must be positive numbers.
- Static reader artifacts remain reproducible from source Markdown.

## Verification

- `npm run typecheck`
- `npx vitest run test/config.test.ts test/render.test.ts test/doctor.test.ts test/cliE2e.test.ts`

## Notes

This covers artifact-size and write-time budgets. Deeper parse, validate,
index, and query timing budgets remain queued in `B005`.
